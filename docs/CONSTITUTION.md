# SteelMind Constitution

Version: 1.0  
Status: **Active (Supreme Authority)**

Evolution note: governance extension is documented in [`CONSTITUTION_V2.md`](./CONSTITUTION_V2.md) while this v1 remains historical baseline.

## 1. Purpose

SteelMind exists to transform technical knowledge into reliable commercial decisions.

The platform must produce estimates that are:

- precise
- traceable
- auditable
- justifiable

SteelMind is not only software. It is an expert system for metallic structures.

## 2. Non-Negotiable Essence

This essence cannot be changed:

> Preserve and operationalize engineering knowledge to generate trustworthy budgets.

Technology can change. Architecture can change. UI can change.  
The knowledge core cannot be lost.

## 3. Mandatory Principles

Every relevant decision must prioritize:

1. Clarity
2. Simplicity
3. Modularity
4. Scalability
5. Testability
6. Observability
7. Security
8. Performance
9. Maintainability
10. Documentation

## 4. Domain-First Rule

The domain is the product.

No technical rule may stay hidden inside UI or ad-hoc scripts.
Every technical rule must include:

- origin
- context
- justification
- version

## 5. AI Grounding Rule

AI must never invent technical facts.

AI reasoning is allowed only over:

- technical standards
- official catalogs
- internal knowledge base
- configured parameters
- explicit user-provided inputs

If data is insufficient, the system must declare uncertainty and request missing inputs.

## 6. Calculation Traceability Rule

No production number is valid without traceability.

Every calculation must expose:

- formula
- formula source
- norm reference
- catalog source
- assumptions
- limitations
- units
- version
- date
- accountable rule owner

Magic numbers are forbidden.

## 7. Architecture Governance

Before structural implementation:

1. Understand the domain
2. Audit current architecture
3. Identify risks and constraints
4. Compare alternatives
5. Record ADR
6. Define migration strategy
7. Implement safely

## 8. Golden Question

For every major decision:

> If we were building today the best metallic-structure budgeting software in the world, would we choose this?

If not, propose a better solution with trade-offs and migration plan.

## 9. Enforcement

Any PR that conflicts with this Constitution must be rejected or rewritten.

This document has precedence over all other architecture notes.
