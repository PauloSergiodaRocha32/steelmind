# ADR-014: Wiki v2 Semantic Cross-links and Traceability Matrix

- Status: **Accepted**
- Date: 2026-07-01

## Context

The Knowledge Platform is active, but discoverability by intent and end-to-end traceability (document -> code -> tests) still depended on manual navigation. This creates onboarding friction and weakens long-term maintainability.

## Decision

Adopt Wiki v2 capabilities generated by docs automation:

1. Semantic cross-links by strategic theme
2. Architecture context map in Mermaid
3. Consolidated risk catalog (roadmap + operational risks)
4. Traceability matrix linking docs to implementation and tests

These artifacts are generated by `npm run bible:index` under `14-Knowledge-Platform`.

## Alternatives considered

1. Keep manual curation only  
   Rejected: high drift risk and low scalability.

2. Move to external docs platform now  
   Rejected: unnecessary migration overhead for current phase.

## Consequences

### Positive

- Faster retrieval for engineers and AI agents
- Better engineering governance and auditability
- Clear path for continuous knowledge evolution

### Negative

- Additional upkeep in docs automation scripts
- Requires discipline to keep mappings aligned with refactors

## Migration / Adoption

1. Add Wiki v2 artifacts in knowledge platform structure
2. Generate artifacts on every documentation cycle
3. Use consolidated risk and traceability matrix in architecture reviews
